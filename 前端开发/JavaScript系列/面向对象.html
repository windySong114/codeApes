面向对象（OOP）

  1. 概念 ​

    • 含义：是一种编程范式，它将程序中的数据和操作封装到对象中

  2. 核心概念 ​

    • 类：类是对象的模板或蓝图，它定义了对象的属性和方法
    • 对象：对象是数据和操作的集合

  3. 三大特征 ​

    • 封装：方法 属性封装到类内部
    • 继承：子类可以继承父类的属性和方法
    • 多态：同一个方法，子类和父类中内容不同

  4. 对象Object ​

    • 原型：每个 JavaScript 对象都有一个原型（prototype），它指向另一个对象。对象可以继承其原型对象的属性和方法。原型对象可以被共享，从而节省内存空间

      js

    继承 ​

    • 原型链继承：在原型链继承中，子类的原型对象指向父类的实例。这意味着子类继承了父类的属性和方法，并且可以沿着原型链访问父类原型上的属性和方法；缺点是所有子类实例共享同一个父类实例，可能导致子类之间的属性相互影响

      js
      function Animal() {
    this.name = 'Animal'
}
Animal.prototype.eat = function() {
    console.log(this.name + ' is eating.')
}
function Dog() {
    this.name = 'Dog'
}
Dog.prototype = new Animal() // 原型链继承
var dog = new Dog()
dog.eat() // 输出: Dog is eating.

    • 构造函数继承：在构造函数继承中，子类在自身的构造函数中调用父类的构造函数，从而实现对父类属性的继承；优点是解决了原型链继承中子类共享父类实例的问题；缺点是无法继承父类原型上的方法

      js
      function Animal(name) {
    this.name = name
}
Animal.prototype.eat = function() {
    console.log(this.name + ' is eating.')
}
function Dog(name) {
    Animal.call(this, name); // 构造函数继承
}
var dog = new Dog('Dog')
dog.eat() // 报错: dog.eat is not a function

    • 组合继承：组合继承结合了原型链继承和构造函数继承的优点。它通过在子类构造函数中调用父类构造函数，并将父类原型赋值给子类原型，实现了同时继承父类属性和方法的功能

      js

Animal.prototype.eat = function() {

}
function Dog(name) {
    Animal.call(this, name) // 构造函数继承
}
Dog.prototype = new Animal() // 原型链继承
var dog = new Dog('Dog')
dog.eat() // 输出: Dog is eating.

  5. 类Class（ES6） ​

    • 含义：是对象的模板，用于创建对象的蓝图

    • 创建与继承

      js
      // class 类名： 通过class创建动物类，Animal为类名
class Animal {
	// 实例成员属性：名称和年龄
	name = ""
	age = 0
	// 静态成员属性：动物的王国
	static kingdom = 'Animalia'
	
    constructor(name, age) {
        this.name = name
		this.age = age
    }
	// 实例成员方法：发出声音
    makeSound() {
        return "Animal sound"
    }
	// 静态成员方法：动物信息
	static info() {
		console.log('This is an Animal class')
	}
}
// class 类名， extends：类的继承关键字
class Dog extends Animal {
	// constructor：构造函数,品种breed：成员属性
    constructor(name, age, breed) {
		// super：调用父类的构造函数
        super(name, age)
        this.breed = breed
    }
	// 实例成员方法，挂载在实例上，通过实例调用，如果实例里没有实例成员方法会通过隐式原型_proto_往prototype链上找：(狗)吠叫
    // 原型：节约内存空间；实现面向对象，避免类型丢失
	bark() {
        return "Woof!"
    }
	// 静态成员方法，挂载在类上，通过类直接调用：对比两只狗的年龄差
	static compareDogs(dog1, dog2) {
		return dog1.age - dog2.age
	}
}
// 创建 Animal 类的实例
let myAnimal = new Animal('Leo', 3)
console.log(myAnimal.name) // 输出: Leo
console.log(myAnimal.age)  // 输出: 3
myAnimal.makeSound()       // 输出: Animal sound
// 调用 Animal 类的静态方法和静态属性
Animal.info();              // 输出: This is an Animal class
console.log(Animal.kingdom); // 输出: Animalia
// 创建 Dog 类的实例
let myDog = new Dog('Buddy', 5, 'Labrador')
console.log(myDog.name)    // 输出: Buddy
console.log(myDog.age)     // 输出: 5
console.log(myDog.breed)   // 输出: Labrador
myDog.makeSound()          // 输出: Animal sound
myDog.bark()              // 输出: Woof!
// 调用 Dog 类的静态方法
let dog1 = new Dog('Max', 4, 'Poodle')
let dog2 = new Dog('Bella', 3, 'Golden Retriever')
console.log(Dog.compareDogs(dog1, dog2)) // 输出: 1 (dog1 比 dog2 年长 1 岁)

    • 使用单例模式把类设计成私有的，可以防止直接实例化

      js
      class Singleton {
    // 私有静态属性，用于存储单例实例
    static #instance = null;
    // 私有构造函数，防止直接实例化
    constructor() {
        // ...
    }
    // 静态方法，用于获取单例实例
    static getInstance() {
        // 如果尚未创建实例，则创建并返回
        if (!Singleton.#instance) {
            Singleton.#instance = new Singleton();
        }
        return Singleton.#instance;
    }
    // 其他方法和属性...
}
// 调用
Singleton.getInstance()
