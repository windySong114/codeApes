对keep-alive的理解

  1. 含义 ​

    • Vue的内置组件，在组件切换过程中可以把状态保留在内存中，防止重复渲染DOM，包裹动态组件时，会缓存不活动的组件实例，而不是销毁他们（缓存组件切换状态和不活动的动态组件实例）

  2. props属性 ​

include匹配的名称，字符串或正则表达式
exclude不匹配的名称，字符串或正则表达式
max最多缓存的组件实例数，数字

  • 示例：

    vue
    <!-- 字符串 -->
<keep-alive include=”a,b”>
    <router-view/>
</keep-alive>
<!-- 正则表达式（使用v-bind） -->
<keep-alive :include="/a|b/">
    <router-view/>
</keep-alive>
<!-- 数组（使用v-bind） -->
<keep-alive :include="['a','b']">
    <router-view/>
</keep-alive>

  1. 生命周期 ​

    • 设置keep-alive缓存组件，会多两个生命周期（activated、deactivated）
    • 首次进入组件时：beforeRouteEnter、beforeCreate、created、mounted、activated、beforeRouteLeave、deactivated
    • 再次进入组件时：beforeRouteEnter、activated、beforeRouteLeave、deactivated

  2. 使用场景 ​

    • 跳转场景：首页->列表页->详情页->返回到列表页（需要缓存）->返回到首页（需要缓存）->再进入列表页（不需要缓存）

    • App.vue设置

      vue
      <!-- App.vue中添加如下代码 -->
<template>
    <div id="app">
        <transition name="fade">
            <keep-alive>
                <router-view v-if="$route.meta.keepAlive" />
            </keep-alive>
            <router-view v-if="!$route.meta.keepAlive" />
        </transition>
    </div>
</template>

    • 路由配置

      js
      // 在路由中配置是否需要缓存的页面
// keepalive为true时需要缓存，为false时不需要缓存
{
    path: '/index',
    name: 'Index',
    component: () => import('@/views/index/index.vue'),
    meta: {
        title: '首页',
        keepalive: true
    }
},
{
    path: '/detail',
    name: 'Detail',
    component: () => import('@/views/index/detail.vue'),
    meta: {
        title: '详情',
        keepalive: false
    }
},

    • 缓存后获取数据：beforeRouteEnter、activated

      vue
      <script>
export default {
    name: 'index',
    data() {
		return {
			msg: "初始数据"
		}
    },
    created() {},
    computed: {},
	beforeRouteEnter(to, from, next) {
		// 无法访问组件实例this
		next(vm => {
			// 可以访问组件实例 `vm`
			vm.loadData();
		});
	},
	activated() {
		this.getData()
	},
	methods: {
		// 更新数据
		loadData() {
		    // 这里可以加载组件需要的数据
		    this.msg = '更新数据';
		},
		// 获取数据
		getData() {
			console.log(this.msg)
		}
	}
}
</script>
