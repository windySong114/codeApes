nextTick是什么

  1. 含义 ​

    • 是一个延迟执行的方法，接收一个回调函数作为参数

      js
      this.$nextTick(() => {
    // 可以访问更新后的 DOM 节点
});
      1
      2
      3

  2. 作用 ​

    • 是一种优化策略，获取异步更新后的DOM内容

  3. 实现原理 ​

    • 通过利用 JavaScript 事件循环机制实现。当 Vue 实例数据发生变化时，Vue 会进行异步更新 DOM

  4. 实现流程 ​

    • 数据变化触发：当修改 Vue 实例的数据时，Vue 不会立即更新 DOM。它会将需要更新的 DOM 记录在一个队列中
    • 等待同步代码执行完毕：Vue 会等待当前的同步代码执行完成。包括当前代码块内的同步任务和微任务
    • DOM 更新：在下一个 microtask 执行时，Vue 会清空队列并且执行实际的 DOM 更新操作
    • nextTick 回调执行：在 DOM 更新完成后，Vue 会执行 nextTick() 中传入的回调函数，可以访问更新后的 DOM 节点

  5. 使用场景 ​

    vue
    <template>
    <div class="page-box" ref="appRef">
        <div class="msg-box">{{ msg }}</div>
        <div @click="onUpdateData">更新数据</div>
    </div>
</template>

<script>
export default {
    name: 'index',
    data() {
        return {
            msg: "初始数据"
        }
    },
    methods: {
        // 更新数据
        async onUpdateData(){
            this.msg = '更新后数据'
            // 输出：“初始数据”，节点数据并非立即更新
            console.log(this.$el.querySelector('.msg-box').textContent) 
            // 节点数据等待nextTick执行完才会更新
            await this.$nextTick()
            // 输出：“更新后数据”
            console.log(this.$el.querySelector('.msg-box').textContent)
        }
    }
}
</script>
